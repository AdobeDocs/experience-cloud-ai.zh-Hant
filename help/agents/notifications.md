# 通知代理

通知代理程式會在[AI小幫手](../ai-assistant/ai-assistant-ui.md)中新增對話式AI層，您可以使用它來使用自然語言來摘要、解譯通知及採取行動，從而將被動摘要變成主動小幫手。 您可以在單一交換中從「我有47個通知」移至「2個需要在星期五前核准，3個資料集發生衝突，其餘為「僅供參考」，而不需掃描長清單。

## 通知代理程式可以執行的動作

通知代理程式支援下列功能：

| 功能 | 說明 |
| --- | --- |
| **通知摘要和深入分析** | 取得通知的簡要概覽，清楚地區分需要採取動作的通知和僅提供資訊的通知，並瞭解通知的目的以及您收到通知的原因。 |
| **操作動作** | 透過對話直接對通知採取行動 — 將其標籤為已讀取、暫停通知，或為個別或分組通知設定提醒。 |
| **喜好設定和設定** | 針對每個解決方案和類別啟用或停用通知通道（電子郵件、Slack、應用程式內），而不需導覽「設定」。 |
| **導覽與深層連結** | 透過深層連結，直接從通知跳至相關的產品區域。 |
| **Collaboration和共用** | 共用通知或傳送公告給相同組織中的收件者。 |

## 存取通知代理程式

您可以透過下列方式存取「通知代理程式」：

- **從AI助理**：開啟AI助理並輸入任何通知相關提示。
- **從提示程式庫**：選取符合您需求的建議提示。

## 範例提示

請閱讀以下章節，以取得可搭配「通知代理程式」使用的提示範例。

### 摘要和深入分析

- 「摘要我最近的通知。」
- 「我的哪些通知需要我的動作，哪些僅供參考？」

### 動作

- &quot;將所有不可操作的通知標示為已讀取。&quot;
- 「一小時後再提醒我有關{NOTIFICATION_OF_INTEREST}通知的事。」

### 偏好設定

- 「啟用Journey Optimizer的Slack通知。」
- 「將我的每日摘要切換為每週。」

### 共同作業

- 「與{EMAIL_ADDRESS_OF_USER_IN_YOUR_ORG}共用此通知。」

## 最佳實務

使用通知代理程式時，請記住下列秘訣：

- **必須具體** — 請清楚命名應用程式、通知、通知類別、頻道，讓代理程式可以在預期的目標上採取行動。
- **鏈結提示** — 您可以要求摘要，接著關注特定通知，然後採取動作，所有這些都在單一交談中。
- **回答澄清問題** — 代理程式要求確認應用程式、頻道或通知時，請務必回應，以便正確套用變更。
